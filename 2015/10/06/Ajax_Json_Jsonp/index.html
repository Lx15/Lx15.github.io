Ajax&Json__Jsonp

一.关于Ajax的基础知识

(1)Ajax是异步的JavaScript和XML，是无需重新加载整个网页的情况下能够更新部分网页的技术，用来进行局部交换。
(2)在没有Ajax之前，客户端的请求是同步的，也就是客户端发送请求到服务端，服务端进行处理、响应，然后发送给客户端，客户端再进行页面加载，在这个过程中客户端会一直等待。在Ajax出现之后，客户端请求变为异步，页面错误会实时显示。在客户端的后台与服务器进行交换数据。之所以有这种变化，是因为Ajax中定义了XMLHttpRequest对象。
(3)在Ajax出现后，实现客户端实时变化依赖于三个部分：

  • 用HTML、CSS实现页面
  • 用XMLHttpRequest和Web服务器进行异步数据交换
  • 用JS操作DOM，实现动态局部刷新

二.关于HTTP的基础知识

(1)HTTP是无状态协议，即Web服务器不保留连接信息，是无记忆的；
(2)一个HTTP请求包括四部分：

  • http请求的方法或动作，比如是GET或者POST方法
  • 请求的地址
  • 请求头
  • 请求体

(3)GET与POST

  • get：一般用于信息的获取，是默认的方法，使用URL传递参数，因此对所有人可见，并且对所发送信息的数量有所限制，至多为2000个字符
  • Post：一般用于修改服务器资源：不在URL中显示，因为对其他人也不可见，数量也无限制，多用发送表单数据。

(4)一个HTTP响应包括三部分：

  • 一个数字和文字组成的状态码用来显示成功还是失败等状态。
    • 1XX表示收到Web浏览器请求，在进步处理
    • 2XX表示成功
    • 3XX表示请求没有成功
    • 4XX表示客户端错误，客户端提交的请求有错误
    • 5XX表示服务器错误。

三.XMLHttpRequest发送请求

  • open(method,url,async) method参数定义发送请求的方法，async默认是true，
  • send(String)将请求发送到服务器，当方法是get时，可以send()或者send(null),使用post方法时不可为空。

四.XMLHttpRequest取得响应

  • responseText：获得字符串形式的响应数据
  • responseXML：获得XML形式的响应数据
  • Status和StatusTex：:以数字和文本形式返回Http状态码
  • getAllResponseHeader()：获取所有的响应报头
  • getResponseHeader()：查询响应中的某个字段中的值
  • readyState：代表服务器的响应的变化
    • 0:请求未初始化，open还未调用
    • 1:服务器连接已建立，open已调用
    • 2:请求已接收，也就是已接收到了头信息
    • 3:请求处理中，也就是接收到响应文本了
    • 4:请求已完成，且响应已就绪，也就是响应已完成

六.Json

1.Json独立于语言，不管什么语言都能解析，可以转换成JavaScript对象
2.Json可用JavaScript内建的方法进行解析，转换成JS对象
3.Json解析两种方法：eval和Json.parse。但eval方法比较危险，较少使用，常用Json.parse方法，此方法出错会进行提醒。
example:


var jsondata=’{“staff”:[{
                      “name”:”xiaoyi”,
                      “age”:alert(“23”);
              }]}’;
var jsonObj=eval(jsondata);
    alert(jsonObj.staff[0].name);
    //结果会先执行alert(“23”),再显示xiaoyi
var jsonObj=JSON.parse(jsondata);
    //结果会报错

七.jQuery实现Ajax

$.ajax([settings]);
1.type:类型 “POST”or”GET”
2.url：发送请求的地址
3.data：对象，和请求发送到服务器的数据
4.dataTye：预期服务器返回的数据类型，如果不指定，jQuery将自动根据HTTP包MIME信息来智能判断，一般采用json格式，可设为“json”
5.succe：方法 请求成功后的回调函数，传入返回后的数据，以及包含成功代码的字符串
6.error：方法 请求失败后执行的函数，传入XMLHttpRequest对象 


$.ajax({
    type:”GET”,
    url:”service.php?number=”+$(“#keyword”).val(),
    dataType：”json”,
    success:function(data){
        if(data.success){
        }
    },
    error:function(jqXHR){
        alert(“error：”+jqXHR.status);
    }
});

八.跨域

  1. Http:// 2. www . 3. abc.com : 4. 8080 / 5. scripts/jquery.js
    1.协议 2.子域名 3.主域名 4.端口号 5.请求资源地址

当协议、子域名、主域名、端口号中任意一个不相同时，都算作不同域。不同域之间相互请求资源，都算作“跨域”。在JS的同源策略下，a.com域名下的JS无法操作b.com或c.com域名下的对象。

九.处理跨域的方法

1.代理
www.a.com/service.php
www.b.com/from-a-service.php //从a处取得服务放在b中，使得b成为代理，前端只需从后端取数据即可
2.Jsonp处理get请求跨域问题
原理是html标签中的script可以进行跨域，但jsonp不能处理post请求的跨域问题 


$.ajax({
    ….
    dataType：”JSONP”,
    jsonp:”calllback123”
});

3.HTML5提供的XMLHttpRequest level2 实现跨域访问及其他功能
header(“Access-Control-Allow-origin:“);//代表所有域都可以访问，也可以设置专门域名
header(“Access-Control-Allow-Methods:POST,GET”);
